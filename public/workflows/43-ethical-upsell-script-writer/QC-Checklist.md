# Quality Control Checklist - Ethical Upsell Script Writer

### 1. Opportunity Identification
- [ ] All products and tiers are documented with clear value propositions
- [ ] Customer journey is mapped from awareness through post-purchase
- [ ] At least 3 natural upsell moments have been identified
- [ ] Each opportunity is ranked by customer benefit alignment
- [ ] Revenue potential is estimated for each opportunity
- [ ] Intrusive or premature upsell moments are flagged and excluded

### 2. Ethical Guardrails
- [ ] Ethical selling mission statement is written and documented
- [ ] Prohibited tactics list is defined (false scarcity, guilt-tripping, etc.)
- [ ] Customer benefit test template is created and ready to apply
- [ ] Transparency standards are documented for pricing and claims
- [ ] Graceful decline protocol is written for each touchpoint
- [ ] Industry-specific compliance requirements are addressed
- [ ] Guardrail document is accessible to everyone who writes or approves scripts

### 3. Script Drafting Quality
- [ ] Each script uses a recognized copywriting framework (PAS, AIDA, etc.)
- [ ] Opening acknowledges the customer's current situation or action
- [ ] Upsell is presented as a solution to a specific customer problem
- [ ] Pricing is stated clearly and early with no hidden conditions
- [ ] Call to action is low-pressure with an easy opt-out
- [ ] Script length is appropriate for placement context
- [ ] Tone matches brand voice and customer expectations
- [ ] Graceful decline response is included for each script

### 4. Claims and Accuracy
- [ ] All benefit claims are specific and measurable
- [ ] Every claim can be substantiated with real data or examples
- [ ] No exaggerated superlatives are used without evidence
- [ ] Social proof references real customer outcomes
- [ ] Pricing includes all fees, conditions, and limitations
- [ ] Refund and cancellation terms are transparent

### 5. Customer Experience
- [ ] Scripts pass the "would I feel good receiving this?" test
- [ ] Decline path is as frictionless as the acceptance path
- [ ] No guilt-inducing language appears anywhere in the script
- [ ] Customer's prior purchase or relationship is acknowledged
- [ ] Upsell timing feels natural, not interruptive
- [ ] Multiple decline options are provided where applicable

### 6. Deployment Readiness
- [ ] Scripts are formatted correctly for each deployment channel
- [ ] Tracking pixels or conversion tracking is configured
- [ ] A/B test variants are prepared for at least one script
- [ ] Scripts are reviewed and approved by at least one other person
- [ ] Deployment schedule is documented
- [ ] Rollback plan exists if scripts receive negative feedback

### 7. Monitoring and Metrics
- [ ] Conversion rate tracking is active for each deployed script
- [ ] Refund rate monitoring is set up for upsold products
- [ ] Customer satisfaction measurement is in place
- [ ] Complaint monitoring channel is established
- [ ] 30-day performance review is scheduled
- [ ] Benchmarks for acceptable performance ranges are defined

### 8. Ongoing Maintenance
- [ ] Quarterly review schedule is set for all active scripts
- [ ] Script update process is tied to product release cycles
- [ ] Underperforming scripts are archived with notes on reasons
- [ ] New upsell opportunities are evaluated monthly
- [ ] Customer feedback is reviewed and incorporated into script updates
- [ ] Ethical guardrails are revisited and updated as needed

### 9. Documentation and Knowledge
- [ ] All scripts are stored in a centralized, organized repository
- [ ] Each script is versioned with change notes
- [ ] Performance data is logged for historical reference
- [ ] Lessons learned are documented after each review cycle
- [ ] Team members are trained on ethical guardrails and script standards
- [ ] Swipe file of top-performing scripts is maintained

### 10. Compliance and Legal
- [ ] Scripts comply with industry advertising regulations
- [ ] Terms and conditions are accurately referenced
- [ ] Pricing representations meet legal standards
- [ ] Customer consent requirements are met for personalized offers
- [ ] Data usage in personalization follows privacy policies
- [ ] Legal review is completed for scripts making income or outcome claims
